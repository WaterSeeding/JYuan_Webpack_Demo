# Webpack 中的 Chunks 是什么？

在 Webpack 中，chunks（组、块）是指打包过程中生成的代码块，它们是模块的集合，可以独立加载。

通过分割代码为不同的 chunks，可以实现按需加载，从而提高应用性能。

## 例子

假设你有一个简单的 Web 应用，包含以下模块:

- index.js (入口文件)
- moduleA.js
- moduleB.js
- moduleC.js

在 index.js 中，你可能只希望在用户点击某个按钮时才加载 moduleB.js 和 moduleC.js。你可以使用动态导入来实现这一点：

```js
// index.js
import("./moduleA").then((module) => {
  module.default();
});

document.getElementById("loadButton").addEventListener("click", () => {
  import("./moduleB").then((module) => {
    module.default();
  });

  import("./moduleC").then((module) => {
    module.default();
  });
});
```

在这个例子中，Webpack 会生成三个 chunks：

1. chunkA - 包含 moduleA.js
2. chunkB - 包含 moduleB.js
3. chunkC - 包含 moduleC.js

当用户加载页面时，只有 chunkA 会被立即加载，而 chunkB 和 chunkC 会在用户点击按钮时异步加载。

这种方式减少了初始加载时间，优化了性能。
